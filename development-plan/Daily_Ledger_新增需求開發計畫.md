## 需求摘要

### R-01：列表頁月份快捷按鈕

- **頁面**：`list.html`
- **位置**：篩選條件區塊內，標題列下方、開始日期標籤列上方
- **新增元素**：`[◀ 上一月]` `[本月]` `[下一月 ▶]` 三個按鈕
- **行為**：
  - 頁面載入預設套用「本月」，自動填入開始日期／結束日期並觸發篩選
  - 點擊任一按鈕 → 填入對應月份區間並立即觸發套用篩選
  - 手動修改日期後，三個按鈕均取消 active 狀態
- **影響**：純前端，`list.html` 單檔修改，無後端異動

---

### R-02：分類管理頁手動排序

- **頁面**：`categories.html`（主要）；`index.html`、`list.html` 下拉選單順序連動
- **UI**：每筆分類列右側加「↑ 上移」「↓ 下移」按鈕；邊界處按鈕 disabled
- **排序範圍**：主類在同類型（E/I）內獨立排序；次類在同主類內獨立排序
- **儲存**：`categories.csv` 新增 `sort_order` 欄位；啟動時自動遷移既有資料
- **新增 API**：`POST /api/categories/reorder`（body：類型、主類、次類、direction）
- **下拉連動**：`GET /api/categories` 改為依 `sort_order` 回傳，所有下拉自動跟隨
- **後置設定**：R-02 實作後直接寫入 `categories.csv` 套用指定初始排序（含補齊缺少分類）
- **影響**：`categories.csv`、`data_manager.py`、`app.py`、`categories.html`

---

### R-03：各頁面標題

- **範圍**：全部六個 HTML 頁面
- **新增元素**：主內容區頂部加頁面標題（Bootstrap Icon + 中文名稱），樣式與既有 UI 一致
- **標題對照**：記帳／交易列表／統計報表／分類管理／匯入／匯出
- **影響**：純前端，六個 HTML 檔，無後端異動

## 實作規劃

### 建議實作順序

| 順序 | 需求 | 理由 |
|------|------|------|
| 1 | R-03（頁面標題） | 最簡單，純 HTML 修改，各頁面獨立，可快速完成 |
| 2 | R-01（月份快捷按鈕） | 純前端，邏輯單一，完成後即可驗收列表頁 |
| 3 | R-02（分類排序） | 最複雜，涉及後端 + 資料遷移 + 前端，最後處理；完成後執行後置排序設定 |

### R-03 實作步驟

1. 確認各頁面 navbar 使用的 Bootstrap Icon class 名稱
2. 在六個 HTML 的主內容區 `<div class="container ...">` 頂部插入 `<h5>` 標題列
3. 瀏覽器逐頁確認標題顯示正常

### R-01 實作步驟

1. 在 `list.html` 篩選區塊加入三個按鈕的 HTML
2. 撰寫 `setMonthRange(offset)` 函式（offset: -1/0/1），計算目標月份首尾日並填入日期欄
3. 綁定按鈕 click 事件：填入日期 → 更新 active 狀態 → 呼叫 `loadList()`
4. 頁面載入時呼叫 `setMonthRange(0)`
5. 監聽開始/結束日期 `change` 事件，清除按鈕 active 狀態

### R-02 實作步驟

1. **後端 data_manager.py**：新增 `sort_order` 欄位遷移邏輯（`init_data_files`）；修改 `get_categories` 依 `sort_order` 排序；新增 `reorder_category` 函式（交換相鄰 sort_order）；`add_category` 自動設末位 sort_order
2. **後端 app.py**：新增 `POST /api/categories/reorder` 端點
3. **前端 categories.html**：主類列與次類列各加「↑ 上移」「↓ 下移」按鈕；click 事件呼叫 API 後重繪列表；邊界按鈕 disabled 邏輯
4. **後置設定**：確認缺少分類（月租費、無謂損失、娛樂交際.淘寶購物），新增後直接修改 `categories.csv` 的 `sort_order` 欄位套用指定排序
