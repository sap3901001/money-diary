## 二、欄位與已知規則

1. 金額欄位位置
   1. DataBase 版本 Main.abd：偏移 0x50（4 bytes）
   2. .mbu 版本 Main.abd：偏移 0x4A（由每筆 284 bytes 的 6-byte 左旋轉導致）

2. 值的性質
   1. 金額欄位為 IEEE 754 單精度 float（LE），但實際上是 **double 的高 4 bytes**（見第五節）
   2. float 儲存的是絕對值
   3. 正負號由交易類型決定：I 為正，E 為負

3. 發票欄位（第一階段已定位）
   1. `Main2.abd` 為 10 bytes 標頭 + 910 筆 × 18 bytes
   2. 每筆 `rec[n][16:18]` 為發票旗標：`0000`=無、`FFFF`=有
   3. 使用者已確認不填入發票號碼，發票號碼文字欄位不列入處理範圍

## 五、定版公式 ✓

### 5.1 編碼原理

MyAB 將金額轉換為「分」（TWD × 100），以 IEEE 754 **雙精度 double** 表示，但**只保留高 4 bytes（big-endian）**，以 little-endian 順序寫入 4-byte 欄位。

等效於：儲存值 = `double(TWD × 100)` 的前 32 bits（含符號位、指數、尾數高位），精度截斷但對整數金額無損。

### 5.2 解碼（stored bytes → TWD）

```python
import struct

def decode_amount(stored_le_4bytes: bytes) -> int:
    """將 Main.abd 0x50 處的 4 bytes (LE) 解碼為 TWD 整數金額"""
    b = stored_le_4bytes
    be_top4 = bytes([b[3], b[2], b[1], b[0]])  # LE → BE
    double_bytes = be_top4 + b'\x00\x00\x00\x00'  # 補零還原 double
    val = struct.unpack('>d', double_bytes)[0]
    return round(val / 100)
```

### 5.3 編碼（TWD → stored bytes）

```python
def encode_amount(twd: int) -> bytes:
    """將 TWD 整數金額編碼為 4 bytes (LE)"""
    double_bytes = struct.pack('>d', twd * 100)
    be_top4 = double_bytes[:4]
    return bytes([be_top4[3], be_top4[2], be_top4[1], be_top4[0]])  # BE → LE
```

### 5.4 正負號規則

- 金額欄位僅儲存**絕對值**
- 交易類型 `E`（支出）→ CSV 輸出為負數
- 交易類型 `I`（收入）→ CSV 輸出為正數
