# money-diary 子專案重構計畫

> 建立日期：2026-04-22

## 背景

`/mnt/c/money-diary/` 下有兩個獨立子專案：

| 子專案 | 說明 |
|--------|------|
| `daily-ledger/` | FastAPI 個人記帳網頁應用，CSV 資料儲存，含完整前端與測試 |
| `myab-export/` | 純 Python 工具，逆向解析 MyAB 記帳簿二進位格式（.abd/.mbu），匯出為 CSV |

兩者邏輯上獨立，myab-export 的輸出可作為 daily-ledger 的匯入來源，但不強制串接。

---

## 重構目標

1. 優化目錄及檔名結構
2. 將開發文件（CLAUDE.md）移出子專案，統一存放於 `development-plan/`
3. 每個子專案有獨立的啟動與關閉腳本
4. 啟動/關閉腳本可在 **Windows cmd** 下直接執行（不依賴 WSL）

---

## 一、daily-ledger 重構

### 1.1 現況

```
daily-ledger/
├── CLAUDE.md                  ← 開發文件，需移出
├── readme.txt
├── requirements.txt
├── .gitignore
├── daily-ledger.code-workspace
├── start.sh / start.bat       ← start.bat 目前依賴 WSL
├── quit.sh / quit.bat         ← 命名不一致（quit vs stop）
├── app.py
├── data_manager.py
├── import_export.py
├── report_engine.py
├── frontend/
├── data/
├── tests/
└── venv/
```

### 1.2 目錄與檔名調整

| 動作 | 原路徑 | 新路徑 |
|------|--------|--------|
| 移出 | `daily-ledger/CLAUDE.md` | `development-plan/daily-ledger_CLAUDE.md` |
| 更名 | `daily-ledger/quit.sh` | `daily-ledger/stop.sh` |
| 更名 | `daily-ledger/quit.bat` | `daily-ledger/stop.bat` |
| 移除 | `daily-ledger/daily-ledger.code-workspace` | 移至根目錄（已有 `money-diary.code-workspace`）|

Python 來源檔維持在根層（`app.py`、`data_manager.py`、`import_export.py`、`report_engine.py`），不移入 `src/`，避免調整 uvicorn 指令與測試 import 路徑。

### 1.3 Windows cmd 啟動腳本（`start.bat`）

重寫為原生 Windows batch，不依賴 WSL：

```batch
@echo off
cd /d %~dp0

if not exist venv\Scripts\python.exe (
    echo 建立虛擬環境...
    python -m venv venv
    if errorlevel 1 (
        echo 錯誤：找不到 Python，請先安裝 Python 3。
        pause
        exit /b 1
    )
    echo 安裝依賴套件...
    venv\Scripts\pip install -r requirements.txt
)

echo 啟動 Daily Ledger...
start http://localhost:8000/
venv\Scripts\python -m uvicorn app:app --port 8000 --reload
```

### 1.4 Windows cmd 關閉腳本（`stop.bat`）

```batch
@echo off
echo 正在關閉 Daily Ledger...
for /f "tokens=5" %%a in ('netstat -ano ^| findstr ":8000"') do (
    taskkill /PID %%a /F >nul 2>&1
)
echo 服務已關閉。
```

> 備註：`stop.bat` 透過終止佔用 port 8000 的 PID 來關閉服務。

### 1.5 WSL/Linux 腳本（`start.sh` / `stop.sh`）

`start.sh` 邏輯維持不變（已支援 WSL 與 Linux）。  
`quit.sh` 更名為 `stop.sh`，內容不變。

### 1.6 重構後結構

```

├── readme.txt
├── requirements.txt
├── .gitignore
├── start.bat              ← 重寫，Windows cmd 原生
├── start.sh               ← 維持（WSL/Linux）
├── stop.bat               ← 重寫（原 quit.bat）
├── stop.sh                ← 更名（原 quit.sh）
├── app.py
├── data_manager.py
├── import_export.py
├── report_engine.py
├── frontend/
├── data/
└── tests/
```

## 二、myab-export 重構

### 2.1 現況

```
myab-export/
├── CLAUDE.md              ← 開發文件，需移出
├── myab-export.py         ← 主程式（一次性執行腳本）
├── readme.txt
├── requirements.txt
├── .gitignore
├── source/                ← 輸入：.abd / .mbu 檔案
└── target/                ← 輸出：CSV 結果
```

無任何啟動/關閉腳本。

### 2.2 目錄與檔名調整

| 動作 | 原路徑 | 新路徑 |
|------|--------|--------|
| 移出 | `myab-export/CLAUDE.md` | `development-plan/myab-export_CLAUDE.md` |

主程式 `myab-export.py` 維持單檔結構，不拆分。  
`source/` 與 `target/` 維持原位。

### 2.3 新增 Windows cmd 啟動腳本（`start.bat`）

myab-export 為一次性腳本（無常駐服務），啟動腳本執行完即結束，無需關閉腳本：

```batch
@echo off
cd /d %~dp0
echo 執行 MyAB 資料匯出...
python myab-export.py
if errorlevel 1 (
    echo 執行失敗，請確認 Python 已安裝並放入 source/ 目錄。
)
echo.
echo 完成。結果存放於 target\ 目錄。
pause
```

### 2.4 新增 WSL/Linux 啟動腳本（`start.sh`）

```bash
#!/bin/bash
cd "$(dirname "$0")"
echo "執行 MyAB 資料匯出..."
python3 myab-export.py
echo "完成。結果存放於 target/ 目錄。"
```

### 2.5 重構後結構

```
myab-export/
├── readme.txt
├── requirements.txt
├── .gitignore
├── start.bat              ← 新增，Windows cmd 原生
├── start.sh               ← 新增，WSL/Linux
├── myab-export.py
├── source/
└── target/
```

## 五、執行順序

### 階段一：myab-export（影響範圍小，先做）
1. 移動 `myab-export/CLAUDE.md` → `development-plan/myab-export_CLAUDE.md`
2. 新增 `myab-export/start.bat`
3. 新增 `myab-export/start.sh`
4. 測試：Windows cmd 下執行 `start.bat`，確認 `target/` 輸出正確

### 階段二：daily-ledger
1. 移動 `daily-ledger/CLAUDE.md` → `development-plan/daily-ledger_CLAUDE.md`
2. 更名 `quit.sh` → `stop.sh`、`quit.bat` → `stop.bat`
3. 重寫 `stop.bat`（Windows cmd 版本）
4. 重寫 `start.bat`（移除 WSL 依賴，改用 Windows Python）
5. 移除 `daily-ledger.code-workspace`
6. 測試：Windows cmd 下執行 `start.bat`，確認瀏覽器開啟、API 正常；執行 `stop.bat` 確認服務關閉

## 六、驗收標準

| 項目 | 驗收方式 |
|------|---------|
| Windows cmd 啟動 daily-ledger | `cd daily-ledger && start.bat`，瀏覽器自動開啟 http://localhost:8000/ |
| Windows cmd 關閉 daily-ledger | `stop.bat`，port 8000 釋放 |
| Windows cmd 執行 myab-export | `cd myab-export && start.bat`，target/ 產生 CSV |
| 開發文件不在子專案目錄 | `daily-ledger/` 與 `myab-export/` 下無 CLAUDE.md |
| 測試通過 | `cd daily-ledger && venv\Scripts\pytest tests/`（Windows）或 `pytest tests/`（WSL）全部通過 |
